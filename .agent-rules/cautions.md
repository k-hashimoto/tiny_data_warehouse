# 注意事項

## 【厳守】役割分担ルール

| 役割 | 担当 |
|---|---|
| プランニング（設計・調査・計画・steering ドキュメント生成） | Claude Code |
| 実装（コード変更・追加・削除） | **Codex MCP のみ** |

- **Claude Code は一切コードを編集しない。`Edit` / `Write` ツールでコードファイルを変更することは禁止**
- **`engineer` サブエージェントも使用禁止。実装は必ず `mcp__codex__codex` ツール（Codex MCP）に委譲する**
- **「一行だけ」「簡単な修正」など規模に関わらず、この役割分担を例外なく適用する**

## 開発フロー厳守事項

- **実装を伴うタスクは規模に関わらず、必ず開発プロセス（planner → 承認 → Codex MCP）に従う。「ダミーでよい」「簡単なタスク」であっても省略しない**
- **Codex MCP を呼び出す前に必ずフィーチャーブランチ（`feature/*`）を作成すること。`staging` ブランチで直接作業しない**

## その他

- ドキュメントの作成・更新は段階的に行い、各段階で承認を得る
- `.steering/` のディレクトリ名は日付と開発タイトルで明確に識別できるようにする
- 永続的ドキュメントと作業単位のドキュメントを混同しない
- コード変更後は必ずリント・型チェックを実施する
- 共通のデザインシステム（Tailwind CSS）を使用して統一感を保つ
- セキュリティを考慮したコーディング（XSS対策、入力バリデーションなど）
- 図表は必要最小限に留め、メンテナンスコストを抑える
