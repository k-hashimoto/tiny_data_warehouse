# Sidebar Guide

The sidebar (Explorer panel) is divided into three sections. This guide explains the features available in each section.

---

## 1. Tables

Displays tables in your main DuckDB database (`app.db`), organized by schema.

### Header buttons

| Button | Action |
|--------|--------|
| **＋** | Create a new schema |
| **↑** | Import a CSV file |
| **↻** | Refresh the table list |

### Working with schemas

- Click the folder icon to expand or collapse a schema.
- Right-click a schema to open the context menu:
  - **Drop Schema (CASCADE)** — deletes the schema and all its tables. The built-in `main` schema cannot be dropped.

### Working with tables

| Action | Result |
|--------|--------|
| Click table name | Expand/collapse column list |
| Double-click table name | Open a new tab with `SELECT * FROM … LIMIT 100` |
| Hover → ℹ︎ icon | Open the Table Metadata modal |
| Right-click → **SELECT * FROM …** | Same as double-click |
| Right-click → **Drop Table** | Delete the table (confirmation required) |

### Column list

Click the chevron (▶) next to a table name to show its columns with their names and data types.

### CSV import (drag and drop)

Drag a `.csv` file onto the Tables section to open the CSV Import dialog.

## 2. dbt

Displays tables generated by [dbt](https://www.getdbt.com/) from `~/.tdwh/db/dbt.db`, organized by schema.

### Header buttons

| Button | Action |
|--------|--------|
| **🔗 / Unlink** | Attach or detach the dbt database. When attached, you can query dbt tables with `FROM dbt.<schema>.<table>` |
| **↻** | Refresh the dbt table list |

### Status badge

- **attached** (green) — dbt.db is attached and queryable.
- **read-only** (blue) — dbt.db is loaded for browsing only.
- **running** (yellow banner) — a `dbt run` is currently in progress.

### Auto-refresh

When `dbt run` completes and writes to `dbt.db`, the Explorer automatically detects the change and refreshes the table list — no manual action needed.

### Working with dbt tables

| Action | Result |
|--------|--------|
| Double-click table name | Open a new tab with `SELECT * FROM dbt."schema"."table" LIMIT 100` |
| Hover → ℹ︎ icon | Open the Table Metadata modal (read-only for dbt tables) |
| Hover → 🗑 icon on schema | Delete the entire schema |
| Hover → 🗑 icon on table | Delete the individual table |

## 3. Saved Queries

Displays your saved SQL scripts, organized in folders.

### Header buttons

| Button | Action |
|--------|--------|
| **↻** | Refresh the script list |

### Script actions

| Action | Result |
|--------|--------|
| Click script name | Load the script into a new tab |
| Hover → ▶ button | Run the script immediately |
| Hover → 💾 button | Overwrite the script with the current editor content |
| Right-click → **Overwrite** | Same as 💾 button |
| Right-click → **Rename** | Edit the script name inline |
| Right-click → **Delete** | Delete the script (confirmation required) |

### Folders

Scripts can be organized in folders. A folder is created automatically when you save a script with a `/` in its name (e.g., `analysis/monthly_revenue`).

### Saving from the editor

Use **Ctrl/Cmd + S** in the SQL editor (or the save button in the editor toolbar) to save the current query as a script. You will be prompted to enter a name; use `/` as a separator to save into a folder.

## Table Metadata

Click the ℹ︎ icon on any table to open the Table Metadata modal.

| Field | Editable |
|-------|----------|
| Table name | — |
| Schema | — |
| Table description | ✓ (app.db tables only) |
| Column names and types | — |
| Column comments | ✓ (app.db tables only) |

dbt table metadata is displayed read-only.
